# 数值溯源规范 (Data Sourcing)

> GregIdle 的核心卖点是「忠实于 GTNH」。任何来自 GTNH 的数值都必须可回溯到锁定版本的源码 / Wiki，**不靠记忆、不靠推测**。
> 版本锚见 [VERSION.md](../VERSION.md)。本文件规定怎么标出处、已锚定哪些数值、变更时怎么复核。

---

## 1. 出处标注格式

### 源码数值（电压、超频、配方机制）
在 `src/data/*.ts` 或设计文档中，紧挨数值写来源注释：

```ts
// 来源：GT5-Unofficial 5.09.52.594 · gt5u-src/.../GTValues.java:93
// V[] = { 8, 32, 128, 512, 2048, ... } EU/t，索引 0=ULV，1=LV
const VOLTAGE_EU_PER_TICK = { LV: 32, MV: 128, HV: 512 /* ... */ };
```

要素：**mod + 版本号 · 相对路径:行号**。行号对应 VERSION.md 锁定的 tag，跨版本会漂移——升级版本时按第 3 节复核。

### Wiki / Dev-Doc 数值（进程、节奏、痛点）
标来源 + 查证日期（Wiki 无历史快照，靠日期）：

```
// 来源：GTNH Steam Age Wiki，查证 2026-06-10：蒸汽机器按 2 L steam = 1 EU 展示消耗
```

### 引用片段
从 `_local/` 源码摘抄的短片段，连同 `文件:行号` 一起，誊抄到使用处的注释或 docs，不整段镜像源码（版权）。

## 3. 数值变更复核流程

当 VERSION.md 锚定的版本升级时：

1. 先更新 VERSION.md 的版本号 / commit / 查证日期。
2. 重新拉取 `_local/gt5u-src/` 等到新 tag，行号可能全变。
3. 逐项核对本文件第 2 节清单：重读源码确认数值未变，更新行号与核实日期。
4. 若数值有变 → 同步改 `src/data/`、设计文档对应章节，并在 STATUS.md 记一笔「数值复核：vX→vY」。
5. 任何核对中发现的源码与设计文档不一致 → 记入 `docs/errors/`。

---

## 4. 防「真实性漂移」

GregIdle 是放置翻译，**允许**简化（如 `getVoltageEffect`），但简化必须：
- 在设计文档明确标为「放置简化」并说明保留了什么核心体验；
- 不冒充成「这就是 GTNH 原值」。

凭记忆写的、查不到出处的 GTNH 数值，一律视为待核实，不得直接进 `src/data/`。
