# 错误记录索引

> 本文件是**指针文档**：每条二级标题指向 `docs/errors/` 下的一个具体错误分析文件。
> 反复出现的问题、踩过的坑、重要教训必须及时记录，避免后续重复找原因。

## 记录规则

- 详细文件命名：`ERROR-YYYY-MM-DD-简述.md`（如 `ERROR-2026-06-19-decimal-serialize.md`）。
- 模板见 [`_TEMPLATE.md`](_TEMPLATE.md)，复制即用。
- 每条至少含五段：现象 / 触发场景 / 根因 / 修复 / 预防。
- 在本文件用二级标题 `## [日期]-[简述]` 加一行摘要，链接到详细文件。
